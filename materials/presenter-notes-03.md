# Presenter notes - Session 3: From SOP to skill

Success metric: everyone leaves with ONE tested skill built from their own SOP.

## Before the session
- Pre-session ask (send 2 days ahead): "bring one written procedure - report format, QA checklist, onboarding steps. Bullets are fine."
- Build one skill from a recognizable company SOP yourself; your live conversion is the opening demo.
- Verify skills are enabled on the Teams workspace (Settings → Capabilities); if org-managed, get one test account confirmed.

## Timing (45)
- 0-3 hook: run YOUR skill live on a real task ("watch it follow all six rules without being told").
- 3-18 concepts: anatomy diagram + description-is-the-doorbell (5) · which-tool-when table (4) · good candidates (2) · pipeline diagram + converter prompt (4). Skill shelf: compress to 3 min if running long; the KOL card lands best with the Karpathy example on screen.
- 18-33 Demo 1 (the main event): pause hard at step 5 (test protocol) - people skip testing and blame the tool later.
- 33-40 Demo 2 (install from shelf).
- 40-45 Q&A + homework (team leads: the SOP-roadmap exercise).

## Watch for
- People pasting SOPs containing client names - remind about the genericize tip BEFORE Demo 1 step 2.
- "My skill didn't fire": it's always the description. Fix one live on the projector - most valuable 90 seconds of the session.
- Engineers asking about scripts/ and Code integration: park to after-session; keep the room non-technical.

## Cuts if long
- Cut Demo 2 to a 60-second presenter-only install. Never cut the test protocol.
